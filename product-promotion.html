Stable Product-Promotion Matching

1. Introduction

Stable Product‚ÄìPromotion Matching uses the Stable Marriage Algorithm to pair products with the most suitable promotions based on mutual preferences. This ensures that no better pairing exists, leading to optimized sales, efficient promotion use, and balanced inventory management ‚Äî ideal for platforms like Amazon.

  • Matches products to promotions based on mutual preferences.
  • Ensures no product‚Äìpromotion pair would prefer each other over their current match (stability).
  • Optimizes promotion impact and sales performance.
  • Prevents misallocation of high-value promotions to low-impact products.
  • Useful in large-scale marketplaces with many products and campaign options.

#include <iostream> #include <vector> #include <climits> #include <string> #include <queue> #include <algorithm> using namespace std; const int N = 3; // Number of products/promotions vector productNames = {"Phone", "Laptop", "TV"}; vector promotionNames = {"Discount10", "HolidayDeal", "Clearance"}; int productPref[N][N] = { {0, 1, 2}, // Phone's preferences {1, 0, 2}, // Laptop's preferences {1, 2, 0} // TV's preferences }; int promotionPref[N][N] = { {1, 0, 2}, // Discount10's preferences {0, 2, 1}, // HolidayDeal's preferences {2, 1, 0} // Clearance's preferences }; int promotionPartner[N]; // Index of matched product bool promotionFree[N]; // Status of each promotion int productNext[N]; // Next promotion to propose to // Returns true if promotion prefers newProduct over currentPartner bool prefers(int promotion, int newProduct, int currentPartner) { for (int i = 0; i < N; ++i) { if (promotionPref[promotion][i] == newProduct) return true; if (promotionPref[promotion][i] == currentPartner) return false; } return false; } void stableMatching() { fill(begin(promotionFree), end(promotionFree), true); fill(begin(promotionPartner), end(promotionPartner), -1); fill(begin(productNext), end(productNext), 0); int freeCount = N; while (freeCount > 0) { for (int p = 0; p < N; ++p) { if (productNext[p] >= N) continue; int promo = productPref[p][productNext[p]++]; if (promotionFree[promo]) { promotionFree[promo] = false; promotionPartner[promo] = p; --freeCount; } else { int current = promotionPartner[promo]; if (prefers(promo, p, current)) { promotionPartner[promo] = p; ++productNext[current]; // Let the old product continue proposing } else { ++productNext[p]; // Current product continues proposing } } } } // Output the matches cout << "Stable Product‚ÄìPromotion Matches:\n"; for (int i = 0; i < N; ++i) { cout << "- " << productNames[promotionPartner[i]] << " ‚áÑ " << promotionNames[i] << "\n"; } } int main() { stableMatching(); return 0; }

4.Real Time Scenario

Stable Product-Promotion Matching is a stepwise process where each product proposes to its preferred promotions in order. Promotions accept the best proposals based on their preferences, possibly replacing current matches. Rejected products continue proposing to other promotions until all matches are stable. This ensures optimal and mutually agreeable pairing of products and promotions for effective marketing.

Initialization

  • proposals[]: Tracks how many times each product has proposed to a promotion (all start at zero).
  • partners[]: Tracks which product is currently matched to each promotion (initially all promotions are unmatched).
  • matched: Number of products currently matched to promotions (starts at zero).

Proposal

Product A proposes to Holiday Electronics Sale.

Promotion Decision

  • If unmatched: Accepts the product immediately.
    • Promotion X is unmatched, accepts Product A.
  • If already matched: Compares current partner with new proposal.
    • If it prefers new product: rejects current and accepts new.
      • Promotion X prefers Product B over Product A. Rejects A, accepts B.
    • If it prefers current partner: rejects new proposal.
      • Promotion X prefers Product A over Product C. Rejects C.

Reaction to Rejection

Product A was rejected, now proposes to Smart Home Discount.

Logging Events

Product A proposed to Holiday Electronics Sale and was accepted.
Promotion X replaced Product A with Product B.
Product C was rejected by Promotion Y.
